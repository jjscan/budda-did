## 1. 시스템 개요

초콜릿 불상 작품 앞에 설치되는 **실시간 가격 전광판(DID)** 시스템.
카메라 비전으로 감지된 **현재 관람 인원 수**에 따라 "한 입" 가격이 실시간으로 등락한다.
관심(사람 수)이 곧 시세가 되는 인터랙티브 아트 장치.

### 인터뷰로 확정된 요구사항

| 항목 | 확정 내용 |
|---|---|
| 가격 변동 요인 | 카메라 비전으로 감지된 **현재 인원 수** (초기 아이디어였던 입구 트리거 센서는 보류) |
| 감지 하드웨어 | 안드로이드 공기계 (삼성 태블릿/스마트폰) 카메라 |
| 시스템 범위 | DID 표시 화면 + 관리자 콘솔 (구매/결제는 범위 외) |
| 사용 맥락 | 실제 전시 운영 |
| 기술 스택 | 순수 HTML/JS 단일 페이지, 안드로이드 기반 웹앱 |
| 네트워크 | 인터넷 가능하나 혼잡 시 매우 불안정 → **오프라인에서도 동작해야 함** |
| 가격 수식 | 상한/하한 + 인원 비례 등락(기본) + 무인 시 미세 움직임(옵션) |
| 표시 방법 | 안드로이드 화면을 TV/모니터에 미러링 |

## 2. 아키텍처 (권장안: 단일 기기 통합형)

```
┌─────────────────── 안드로이드 태블릿 1대 ───────────────────┐
│                                                              │
│  [전면/후면 카메라] → 브라우저 내 사람 감지 (TF.js)          │
│         ↓                                                    │
│  [인원 카운트] → 스무딩(EMA) → [가격 엔진] → [DID 렌더링]   │
│         ↑                                       ↓            │
│  [관리자 콘솔] (숨김 제스처로 진입, localStorage 저장)       │
│                                                              │
└──────────────────────┬───────────────────────────────────────┘
                       │ 화면 미러링 (Smart View 또는 USB-C→HDMI)
                       ▼
                 [전시장 TV / 모니터]
```

**한 대의 태블릿에서 감지→계산→표시를 모두 처리**하고 화면만 TV로 미러링한다.

- 기기 간 통신이 아예 없으므로 **전시장 인터넷이 끊겨도 무관** (최대 리스크 제거)
- 서버·앱 개발 없이 웹페이지 하나로 완결 → 가성비 최상
- 태블릿을 작품 근처에 관람객 방향으로 거치하면 카메라가 "현재 작품 앞 인원"을 감지 — "감지된 인원 수" 요구사항과 정확히 일치

### 확장안 (2기기 분리형 — 필요 시 2차 확장)

입구 감지 등 카메라 위치와 화면 위치를 분리해야 할 경우:
스마트폰(감지 노드) → Firebase Realtime DB → 태블릿(표시).
전송량이 초당 정수 1개 수준이라 느린 인터넷에서도 견디며,
끊김 시 표시 기기는 마지막 카운트 + 미세 움직임 모드로 자연스럽게 유지(fallback).
**1차 구축에서는 제외**하되 코드 구조상 카운트 소스를 교체 가능하게 설계한다.

### 배포 방식: PWA (중요)

브라우저에서 카메라를 쓰려면 HTTPS가 필수(getUserMedia 보안 정책).
→ **GitHub Pages(무료 HTTPS)에 호스팅 + Service Worker로 전체 캐싱(PWA)**

- 전시 전 와이파이에서 1회 로드하면 이후 **완전 오프라인 동작**
- 감지 모델 파일까지 캐싱하여 재방문·재부팅 시에도 네트워크 불필요
- 홈 화면에 추가하면 주소창 없는 전체 화면 실행

## 3. 사람 감지 방식 비교 (미정 항목 → 권장안 제시)

| 항목 | ① 브라우저 기반 (TF.js/MediaPipe) ★권장 | ② 안드로이드 네이티브 앱 (ML Kit) |
|---|---|---|
| 개발 공수 | 웹페이지에 라이브러리 추가만 | 별도 앱 개발 + 기기 설치·서명 배포 |
| 스택 통일성 | 순수 HTML/JS 하나로 완결 (요구사항 부합) | 웹 + 안드로이드 두 스택 관리 |
| 인식 성능 | 최신 삼성 기기 기준 10~30fps | 더 빠르고 안정적 |
| 다인원 카운트 | COCO-SSD person 클래스: 제한 없음 | 지원 |
| 표시 화면과 통합 | 같은 페이지에서 처리 (통신 불필요) | 앱→웹 간 통신 계층 필요 |
| 유지보수 | URL 갱신만으로 업데이트 | 앱 재설치 필요 |

**판단**: 이 시스템에 필요한 것은 초당 0.5~1회의 "현재 몇 명" 카운트뿐이다.
고정밀 추적이 불필요하므로 브라우저 기반 성능으로 충분하며,
전체 시스템을 순수 HTML/JS 단일 페이지로 통일할 수 있는 ①을 권장한다.

- 감지 모델: **TensorFlow.js + COCO-SSD** (person 클래스 필터) 기본,
  성능 미달 시 MediaPipe Object Detector로 교체 (인터페이스 분리 설계)
- 카운트 안정화: 지수이동평균(EMA) + 히스테리시스로 순간 오인식에 의한 가격 널뛰기 방지

## 4. 가격 엔진 사양

```
가격(t) = clamp( 기준가 + 인원수(EMA) × 인당단가 + 미터기 누적 가산액, 하한, 상한 )
```

| 파라미터 | 설명 | 예시 기본값 |
|---|---|---|
| 기준가 | 무인 상태의 기본 가격 | ₩2,500,000 |
| 인당단가 | 감지 인원 1명당 가산액 | ₩70,000 |
| 상한 / 하한 | 가격 허용 범위 | ₩1,000,000 ~ ₩9,900,000 |
| 무인 가산 (택시 미터기) | 무인 상태에서 카운트다운이 흐르고 0 도달 시 가산액 누적. 관람객 등장 시 카운트다운 리셋 | 30초마다 +₩10,000 |
| 갱신 주기 | 화면 가격 갱신 간격 | 2초 |
| 전환 연출 | 목표가로 틱 단위 애니메이션 + 등락 색 플래시 | 상승 빨강 유지 |

- 인원이 줄면 인원 비례분은 내려가지만, 미터기 누적 가산액은 유지됨 (택시 요금처럼 단방향)
- 누적 가산액은 localStorage에 보존(재부팅에도 유지), 관리자 콘솔의 "미터기 가산액 초기화"로 리셋
- 상한 도달 시 미터기 가산 정지
- 카운트다운은 우상단 ⚙ 왼쪽에 은근하게 표시(옵션, 기본 ON): `무인 가산 +₩10,000 (00:19)` 형식 — 운영자용, 관람객 눈에 안 띄는 어두운 색
- DID 화면에 실시간 가격 차트 표시: 종류 선택 가능(①캔들 ②꺾은선), 봉 주기(기본 5분)와 X축 범위(기본 1440분=1일)를 설정에서 독립 조정 — 봉 개수 = 범위 ÷ 주기(1일 기본 288봉, 상한 2000), 상승 빨강/하락 파랑(한국 시세 관행), 마지막 봉 실시간 갱신, 현재가 점선 + 라벨, 봉 데이터는 localStorage에 보존되어 재부팅 시 이어 그림(주기 변경 시 리셋, 범위 변경은 리셋 없음)
- 모든 파라미터는 localStorage에 저장되어 재부팅 후에도 유지
- (변경 이력) 초기 설계의 "무인 시 랜덤 미세 변동"은 택시 미터기 방식으로 대체됨 (2026-07-05)

## 5. 화면 구성

### 5-1. DID 표시 화면 (2단 시세판 레이아웃 — 2026-07-05 목업 반영)

- 검정 배경, **가로형 2단 레이아웃** (TV 미러링 기준, 세로 화면은 자동으로 한 열 스택)
- **좌측**: 타이틀 → `한 입에` + 붉은 테두리 박스 안 도트 매트릭스 LED 가격(등락 틱 애니메이션) → 캔들차트(가격(₩) 캡션·좌측 가격 눈금·하단 시각 라벨·현재가 점선) → 하단 멘트
- **우측**: `현재 가격`(붉은 강조 + ▲▼ 등락액·등락률, 오늘 시가 대비) / `현재 참여 인원` / `오늘 최고가` / `오늘 최저가` — 구분선으로 분리
- 오늘 시가·최고가·최저가는 날짜별로 localStorage에 보존 (재부팅 유지, 날짜 바뀌면 자동 리셋, 관리자 콘솔에서 수동 리셋 가능)
- 타이틀·단위 문구·하단 멘트는 관리자 콘솔에서 자유 편집 (localStorage 저장)
- 화면 꺼짐 방지 (Wake Lock API)

### 5-2. 관리자 콘솔 (같은 페이지 내 숨김 패널) — 구현 완료

- 진입: 우상단 ⚙ 버튼 또는 화면 아무 곳 더블 탭 (+ 선택적 PIN 잠금, 비우면 잠금 없음, 인증은 새로고침 전까지 유지)
- 가격 파라미터 전체 조정 (기준가·인당단가·상한하한·미터기·주기·차트)
- 문구 편집 (타이틀·단위·하단 멘트)
- **캘리브레이션 뷰**: 카메라 프리뷰 + 감지 박스(신뢰도 %) + 실시간 카운트 표시 (거치 각도 조정용)
- 현재 상태 모니터: 카운트 소스, 감지 인원, EMA, 미터기 누적, 오늘 시가, 목표가, 표시가
- 가격 고정 모드 (시연·촬영용 — 인원·미터기 무시, 미터기 정지)

### 5-3. 배포 (P3 완료)

- 저장소: https://github.com/jjscan/budda-did (공개)
- 배포 주소: **https://jjscan.github.io/budda-did/**
- PWA: manifest.json(fullscreen·landscape) + sw.js(앱 셸 프리캐시, 페이지는 네트워크 우선, TF.js·모델 파일은 캐시 우선) + LED ₩ 아이콘 192/512
- 오프라인 절차: 전시 전 온라인에서 한 번 열고 카메라를 켠다 → 라이브러리·모델까지 캐시됨 → 이후 오프라인 재기동 가능
- 업데이트 배포: `git push` 후 Pages 자동 반영 (수 분 소요), 재방문 시 새 버전 수신
